## evo-foundation-models-ai-development-workshop

## Описание

Шаблон для AI-реализации API для анализа настроений на базе ИИ, созданный с использованием NestJS, который предоставляет интеллектуальные возможности анализа настроений через модели LLM, совместимые с OpenAI. Сервис анализирует текстовые сообщения и возвращает структурированные JSON-ответы с классификацией эмоций (приятное, нейтральное, негативное) и уровнями уверенности, сохраняя при этом полный журнал всех обменов данными.

Для реализации установите расширение Roo Code и выполните задачу из файла task.txt сначала в режиме архитектора, следом в режиме разработки.
Инструкция для настройки Roo Code доступна на сайте cloud.ru https://cloud.ru/docs/tutorials-evolution/list/topics/foundation-models__connect-vscode

Детали реализации проекта можно посмотреть в вебинаре https://cloud.ru/events/ai-razrabotka-pod-kontrolem-kak-sokratit-tekhdolg-i-uiazvimosti-11347437637

## Возможности

- **Анализ настроений на базе ИИ**: Отправка текстовых сообщений в настраиваемые модели LLM, совместимые с OpenAI
- **Структурированные JSON-ответы**: Получение оценок эмоций (приятное | нейтральное | негативное) с уровнями уверенности (0-1)
- **Полный журнал аудита**: Все пары запрос-ответ сохраняются в базе данных
- **Аутентификация на основе токенов**: Безопасный доступ к API с использованием заголовка X-Auth-Token
- **Сохранение в базе данных**: Полное логирование промптов, ответов, эмоций, уверенности, задержки и статуса HTTP
- **Мониторинг работоспособности**: Встроенные конечные точки проверки работоспособности для мониторинга системы
- **Документация Swagger**: Автоматически генерируемая документация API

## Конечные точки API

### POST /ai/message
Отправить текст для анализа настроений и получить ответ ИИ.

**Аутентификация**: Требуется (заголовок X-Auth-Token)

**Тело запроса**:
```json
{
  "text": "Ваше текстовое сообщение для анализа"
}
```

**Ответ**:
```json
{
  "messageId": "uuid",
  "emotion": "pleasant | neutral | negative",
  "confidence": "number",
  "createdAt": "date-time"
}
```

### GET /health
Проверить состояние работоспособности приложения и подключение к базе данных.

**Ответ**:
```json
{
  "status": "ok",
  "timestamp": "date-time",
  "uptime": "number",
  "environment": "string",
  "database": {
    "status": "connected",
    "name": "string",
    "host": "string",
    "port": "number",
    "version": "string",
    "connections": {
      "current": "number",
      "available": "number"
    }
  }
}
```

## Конфигурация

Приложение использует переменные окружения для конфигурации. Скопируйте `env.example` в `.env` и настройте следующие переменные:

### Приложение
- `NODE_ENV`: Среда (development/production)
- `PORT`: Порт приложения (по умолчанию: 3000)

### База данных
- `MONGODB_URI`: Строка подключения к MongoDB

### Конфигурация LLM
- `LLM_BASE_URL`: Базовый URL для сервиса LLM, совместимого с OpenAI
- `LLM_API_TOKEN`: Токен API для аутентификации в сервисе LLM
- `LLM_MODEL`: Имя модели для анализа настроений

### Аутентификация
- `API_TOKEN`: Токен для аутентификации API (используется в заголовке X-Auth-Token)

## Аутентификация

API использует аутентификацию на основе токенов. Включите следующий заголовок в ваши запросы:

```
X-Auth-Token: your-api-token
```

Значение токена должно соответствовать переменной окружения `API_TOKEN`.

## Модель данных

Все запросы и ответы анализа настроений сохраняются в базе данных со следующей структурой:

```json
{
  "messageId": "uuid",
  "requestText": "string",
  "emotion": "string",
  "confidence": "number",
  "latencyMs": "number",
  "statusCode": "integer",
  "createdAt": "date-time"
}
```

## Настройка Docker для разработки

Этот проект включает конфигурацию Docker Compose для запуска MongoDB во время разработки.

### Предварительные требования

- [Docker](https://www.docker.com/get-started)
- [Docker Compose](https://docs.docker.com/compose/install/)

### Разработка с Docker

Для запуска приложения с MongoDB в Docker:

```bash
# Запуск MongoDB с помощью Docker Compose
$ yarn run compose

# Запуск приложения в режиме разработки
$ yarn run start:dev
```

### Ручное управление Docker

Если вам нужен больший контроль над контейнерами Docker:

```bash
# Запуск сервиса MongoDB
$ docker-compose up -d mongodb

# Остановка сервиса MongoDB и удаление томов
$ docker-compose down -v

# Просмотр логов
$ docker-compose logs mongodb
```

### Конфигурация

Настройка Docker использует следующую конфигурацию:

- Версия MongoDB: Последняя стабильная версия
- Имя базы данных: Настраивается через переменную окружения MONGODB_URI
- Порт: 27017 (проброшен на хост)
- Том данных: Постоянный том для хранения данных

Переменные окружения определены в файле `.env`.
